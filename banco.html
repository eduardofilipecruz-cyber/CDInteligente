Fundamentos de Banco de Dados

Conceitos essenciais de SQL e modelagem de dados.

SQL

Explicação
SQL (Structured Query Language) é a linguagem padrão para consultar e manipular dados em bancos relacionais.
Exemplo
SELECT * FROM alunos;
Curiosidade
SQL surgiu na IBM nos anos 1970 e até hoje é o padrão mais usado em bancos relacionais.

INSERT

Explicação
O comando INSERT adiciona novos registros em uma tabela.
Exemplo
INSERT INTO alunos (nome, idade)
VALUES ('João', 20);
Curiosidade
É possível inserir vários registros em um único comando, separando os valores por vírgula.

SELECT

Explicação
SELECT é usado para consultar dados, podendo filtrar com WHERE e ordenar com ORDER BY.
Exemplo
SELECT nome, idade FROM alunos
WHERE idade > 18
ORDER BY nome;
Curiosidade
O asterisco * seleciona todas as colunas, mas em produção é recomendado especificar apenas as necessárias, por desempenho.

UPDATE

Explicação
UPDATE modifica registros existentes. É essencial usar WHERE para não alterar a tabela inteira.
Exemplo
UPDATE alunos
SET idade = 21
WHERE id = 5;
Curiosidade
Esquecer o WHERE em um UPDATE é um dos erros mais perigosos em produção: atualiza todas as linhas.

DELETE

Explicação
DELETE remove registros de uma tabela, também exigindo uma condição segura com WHERE.
Exemplo
DELETE FROM alunos
WHERE id = 5;
Curiosidade
Muitos sistemas preferem "exclusão lógica" (marcar como inativo) em vez de excluir o dado de fato.

Modelagem

Explicação
Modelar um banco é planejar entidades, atributos e relacionamentos antes de criar as tabelas, geralmente usando diagramas ER.
Exemplo
Aluno (id, nome, turma_id)
Turma (id, nome)
-- Aluno pertence a uma Turma
Curiosidade
Boa modelagem evita redundância de dados, processo conhecido como normalização.

SQLite

Explicação
SQLite é um banco de dados leve, sem servidor, armazenado em um único arquivo — muito usado em apps e protótipos.
Exemplo
CREATE TABLE alunos (
  id INTEGER PRIMARY KEY,
  nome TEXT NOT NULL
);
Curiosidade
O SQLite está embutido em praticamente todo smartphone, sendo um dos bancos de dados mais usados do mundo.

Relacionamentos

Explicação
Tabelas se relacionam por chaves estrangeiras (foreign keys), formando vínculos como 1:1, 1:N e N:N.
Exemplo
CREATE TABLE matriculas (
  aluno_id INTEGER,
  turma_id INTEGER,
  FOREIGN KEY(aluno_id) REFERENCES alunos(id),
  FOREIGN KEY(turma_id) REFERENCES turmas(id)
);
Curiosidade
Relacionamentos N:N geralmente exigem uma tabela intermediária, chamada tabela associativa.
