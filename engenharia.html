Fundamentos de Engenharia de Software

Como planejar e organizar um projeto de software com qualidade.

Planejamento

Explicação
Planejamento define escopo, prazos e recursos antes de iniciar o desenvolvimento, reduzindo riscos e retrabalho.
Exemplo
1. Definir objetivo do projeto
2. Listar entregas
3. Estimar prazos
4. Definir responsáveis
Curiosidade
Metodologias ágeis, como Scrum, planejam em ciclos curtos chamados sprints, em vez de um plano único e fixo.

Levantamento de requisitos

Explicação
É o processo de identificar o que o sistema precisa fazer (requisitos funcionais) e como deve se comportar (requisitos não funcionais).
Exemplo
Requisito funcional: "O sistema deve permitir login."
Requisito não funcional: "O login deve responder em até 1 segundo."
Curiosidade
Requisitos mal levantados são apontados como uma das principais causas de falha em projetos de software.

Diagrama simples

Explicação
Diagramas ajudam a visualizar fluxos e estruturas do sistema antes de codificar, como um diagrama de fluxo de login.
Exemplo
[Usuário] -> [Tela de Login] -> [Validação] -> [Dashboard]
                                     |
                                 [Erro] -> [Mensagem de erro]
Curiosidade
A notação UML é um padrão internacional para representar diagramas de software de forma universal.

Organização do projeto

Explicação
Organizar pastas, arquivos e nomes de forma clara facilita a manutenção e o trabalho em equipe.
Exemplo
/css   -> estilos
/js    -> lógica
/img   -> imagens
index.html -> entrada do sistema
Curiosidade
Muitas empresas adotam padrões de pastas (como MVC) para que qualquer desenvolvedor entenda o projeto rapidamente.

Boas práticas

Explicação
Boas práticas incluem nomear variáveis claramente, evitar duplicação de código e seguir um padrão de formatação consistente.
Exemplo
// Ruim
let x = 10;

// Bom
let quantidadeDeAlunos = 10;
Curiosidade
O princípio DRY ("Don't Repeat Yourself") é uma das regras mais citadas em engenharia de software.

Documentação

Explicação
Documentar é registrar decisões, funcionamento e instruções de uso do sistema, facilitando manutenção futura.
Exemplo
# Como instalar
1. Baixe o projeto
2. Abra o index.html no navegador
Curiosidade
Um README bem escrito pode reduzir drasticamente o tempo que um novo desenvolvedor leva para entender um projeto.
