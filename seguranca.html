🔐 Segurança

Fundamentos de Segurança

Boas práticas para proteger sistemas, dados e usuários.

Login

Explicação
O login é o processo de identificar e autorizar um usuário, normalmente por usuário e senha, antes de liberar o acesso ao sistema.
Exemplo
Usuário digita usuário e senha
Sistema verifica se os dados existem e coincidem
Se sim -> acesso liberado
Se não -> mensagem de erro
Curiosidade
Sistemas modernos frequentemente combinam login com autenticação em duas etapas (2FA) para reforçar a segurança.

Senhas

Explicação
Senhas fortes combinam letras, números e símbolos, evitando dados óbvios como datas de nascimento.
Exemplo
Fraca:  123456
Forte:  Tr0p1c0!2026#
Curiosidade
Em sistemas reais, senhas nunca devem ser armazenadas em texto puro — devem passar por funções de hash, como bcrypt.

Validação

Explicação
Validar dados de entrada evita erros e ataques, garantindo que apenas informações esperadas sejam processadas.
Exemplo
if (usuario === "" || senha === "") {
  mostrarErro("Preencha todos os campos");
}
Curiosidade
A validação deve ocorrer tanto no front-end (experiência do usuário) quanto no back-end (segurança real).

Proteção de dados

Explicação
Proteger dados envolve controlar quem acessa quais informações e garantir que dados sensíveis não sejam expostos.
Exemplo
Mostrar apenas: "Olá, João"
Nunca expor: senha, token interno, dados de outros usuários
Curiosidade
Leis como a LGPD (Brasil) e a GDPR (Europa) exigem que empresas protejam formalmente os dados pessoais dos usuários.

Boas práticas

Explicação
Boas práticas de segurança incluem nunca confiar apenas no front-end, limitar tentativas de login e manter dependências atualizadas.
Exemplo
- Limitar tentativas de login
- Expirar sessões inativas
- Nunca expor senhas em logs
Curiosidade
A maioria dos ataques bem-sucedidos explora falhas simples e já conhecidas, não técnicas avançadas.

Autenticação

Explicação
Autenticação confirma a identidade do usuário; autorização define o que ele pode fazer depois de autenticado.
Exemplo
Autenticação: "Você é realmente o usuário João?"
Autorização: "João pode editar este relatório?"
Curiosidade
Autenticação e autorização são conceitos diferentes, mas frequentemente confundidos até por desenvolvedores experientes.
